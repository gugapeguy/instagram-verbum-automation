## Story

**Como** CEO Operações,  
**quero** um módulo centralizado que valide a presença das credenciais da Meta Graph API antes de qualquer operação,  
**para que** erros de configuração sejam detectados no início do fluxo, com mensagens claras de como resolver, sem nunca expor as credenciais ao repositório.

3. `api/config.js` exporta: `{ pageId, accessToken, appId, appSecret, baseUrl }` onde `baseUrl = 'https://graph.facebook.com/v18.0'`
4. `.env.example` criado na raiz com as 4 variáveis comentadas (sem valores reais) e instruções de como obter cada valor
5. `.gitignore` verificado: `.env` listado — se não estiver, adicionar
6. Função `testConnection()` exportada: faz GET em `${baseUrl}/${pageId}?fields=name&access_token=${accessToken}`, retorna `{ connected: true, page_name: string }` em caso de sucesso ou `{ connected: false, error: string }` em caso de falha

## Dev Notes

**Estrutura de `api/config.js`:**
```javascript
require('dotenv').config();

const required = ['META_PAGE_ID', 'META_ACCESS_TOKEN', 'META_APP_ID', 'META_APP_SECRET'];

for (const key of required) {
  if (!process.env[key]) {
    throw new Error(
      `${key} ausente. Configure o arquivo .env na raiz do projeto. ` +
      `Veja .env.example para referência de como obter este valor.`
    );
  }
}

const config = {
  pageId: process.env.META_PAGE_ID,
  accessToken: process.env.META_ACCESS_TOKEN,
  appId: process.env.META_APP_ID,
  appSecret: process.env.META_APP_SECRET,
  baseUrl: 'https://graph.facebook.com/v18.0',
};

async function testConnection() {
  try {
    const res = await fetch(`${config.baseUrl}/${config.pageId}?fields=name&access_token=${config.accessToken}`);
    const data = await res.json();
    if (data.error) return { connected: false, error: data.error.message };
    return { connected: true, page_name: data.name };
  } catch (err) {
    return { connected: false, error: 'Falha de rede ao conectar com o Graph API' };
  }
}

module.exports = { ...config, testConnection };
```

**Conteúdo de `.env.example`:**
```
# Meta Graph API — configurações necessárias para publicação automática
# Obter em: https://developers.facebook.com > Seu App > Graph API Explorer

META_PAGE_ID=           # ID numérico da sua Página do Facebook
META_ACCESS_TOKEN=      # Page Access Token (permissões: instagram_basic, instagram_content_publish)
META_APP_ID=            # App ID do seu App no Meta for Developers
META_APP_SECRET=        # App Secret do seu App no Meta for Developers

# ATENÇÃO: Este arquivo (.env) NUNCA deve ser commitado. Já está no .gitignore.
```

**Dependência npm:** `dotenv` — verificar se já está instalado. Se não: `npm install dotenv`.

**CRÍTICO:** `.env` nunca commitado. Se o arquivo `.env` for criado localmente para testes, confirmar que `.gitignore` já o ignora antes de qualquer `git add`.

2. **Com `.env` válido:** `node -e "const c = require('./api/config'); console.log(c.pageId)"` → deve imprimir o valor
3. **`.gitignore`:** `git status` depois de criar `.env` → `.env` deve aparecer como "ignored", não como "untracked"
4. **`testConnection()`:** `node -e "require('./api/config').testConnection().then(console.log)"` → `{ connected: true, page_name: '...' }`
