## Story

**Como** Gustavo,  
**quero** que o CEO Master entregue o pacote de aprovação sempre no formato exato definido no CLAUDE.md,  
**para que** eu possa aprovar com uma única resposta (`aprovado · [horário] · [A ou B]`) e a equipe execute a publicação sem precisar me acionar novamente.

## Acceptance Criteria

1. CEO Master monta o pacote de aprovação sempre no formato canônico definido no CLAUDE.md, sem desvios
2. Seção COPY: legenda completa humanizada (output da Story 2.2), pronta para copiar e colar
3. Seção HASHTAGS: bloco formatado com os 15 hashtags (5+5+5 da Story 2.2)
4. Seção DESIGN A: referência ao HTML gerado pelo CEO Criativo com thumbnail descritivo e link/descrição
5. Seção DESIGN B: prompt completo para Canva manual (instrução suficiente para recriar sem ajuda)
6. Instrução de aprovação clara ao final: `Para aprovar: aprovado · [horário] · [A ou B]`
7. Tempo total do comando de Gustavo até entrega do pacote: ≤ 5 minutos
8. CEO Operações SÓ age após receber `aprovado` explícito com horário e escolha de design (A ou B)

## Dev Notes

**Formato canônico — fonte da verdade:**  
O formato está definido no `CLAUDE.md` da raiz do projeto (seção "Pacote de Aprovação"). Qualquer divergência é um bug. O @dev deve reproduzir EXATAMENTE.

**Formato canônico do CLAUDE.md:**
```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
ENTREGA — @verbumcontabilidade
[tipo] · [pilar] · [data sugerida]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

COPY
[legenda completa]

HASHTAGS
[bloco de hashtags]

DESIGN A — Canva MCP
[thumbnail] · [link Canva]
Status: ✅ pronto / ⚠️ revisar

DESIGN B — Prompt para Canva manual
[prompt completo]

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Para aprovar: aprovado · [horário] · [A ou B]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

**Nota sobre "Design A — Canva MCP":**  
O label do CLAUDE.md diz "Canva MCP" mas a Regra 10 do CLAUDE.md diz: "CEO Criativo usa HTML/CSS via Claude como ferramenta principal. Canva MCP descartado." Portanto:
- **Design A** = HTML/CSS gerado pelo CEO Criativo → manter label como "DESIGN A — HTML/CSS" e incluir link para o arquivo HTML ou a thumbnail
- **Design B** = prompt completo para recriar manualmente no Canva (fallback manual)

**Design B — o que é um prompt Canva completo:**
```
Prompt Canva Manual:
- Dimensões: 1080x1080px
- Fundo: Navy #0D1B3E
- Slide 1: Título "O que é o IBS?" em Cormorant Garamond 700, cor Gold #C9A84C, centralizado
- Slide 2: Corpo em Outfit 300, branco, bullet points com 3 pontos principais
- [... instrução completa slide por slide]
- Logo Verbum: canto inferior direito, variante clara
```

**Gate de aprovação — validação:**
Resposta válida de Gustavo: `aprovado · 09:00 · A`  
Variações aceitas: `aprovado · 18h30 · B`, `aprovado · amanhã 8h · A`  
Resposta inválida: `aprovado`, `ok`, `pode publicar` (sem horário/design) → pedir confirmação

**Rastreabilidade do fluxo:**  
Após cada pacote entregue, o CEO Master salva em `logs/publicacoes.json`:
```json
{
  "data_comando": "2026-06-09T10:00:00",
  "tema": "IBS e o impacto no Simples Nacional",
  "pilar": 1,
  "formato": "carrossel",
  "status": "aguardando_aprovacao"
}
```
