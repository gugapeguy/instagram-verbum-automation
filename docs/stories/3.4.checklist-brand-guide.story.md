## Acceptance Criteria

1. `skills/brand-guide-checker.md` criada documentando: propósito, critérios de validação, como usar, output esperado
2. Critério de paleta navy: HTML contém `#0D1B3E` ou `#081530` ou `#132860` — pelo menos uma cor navy presente
3. Critério de paleta gold: HTML contém `#C9A84C` ou `#E8C97A` — pelo menos uma cor gold presente
4. Critério de tipografia: HTML referencia `Cormorant` e `Outfit` (Google Fonts link ou CSS font-family)
5. Critério de logo: HTML contém elemento com classe `logo-verbum` ou img com `logo` no src
6. Status de saída `✅ APROVADO — brand guide conforme` quando todos os critérios passam
7. Status de saída `⚠️ REVISAR — [lista de critérios falhados]` quando um ou mais critérios falham
8. Design A NUNCA avança para o pacote de aprovação com status `⚠️ REVISAR` sem revisão manual explícita
9. Tempo de execução do checker: < 5 segundos

## Dev Notes

**Lógica de validação (pseudocódigo):**
```javascript
function checkBrandGuide(htmlContent) {
  const results = [];
  
  const hasNavy = ['#0D1B3E', '#081530', '#132860'].some(c => htmlContent.includes(c));
  if (!hasNavy) results.push('paleta navy ausente');
  
  const hasGold = ['#C9A84C', '#E8C97A'].some(c => htmlContent.includes(c));
  if (!hasGold) results.push('paleta gold ausente');
  
  const hasTypo = htmlContent.includes('Cormorant') && htmlContent.includes('Outfit');
  if (!hasTypo) results.push('tipografia brand guide ausente (Cormorant Garamond + Outfit)');
  
  const hasLogo = htmlContent.includes('logo-verbum') || /logo/i.test(htmlContent);
  if (!hasLogo) results.push('logo Verbum ausente');
  
  if (results.length === 0) {
    return '✅ APROVADO — brand guide conforme';
  }
  return `⚠️ REVISAR — ${results.join('; ')}`;
}
```

**Mensagem de escalation ao CEO Master (quando REVISAR):**
```
⚠️ Brand Guide Check — REVISAR necessário
Design A gerado para: [tema]
Issues: [lista dos critérios falhados]
Ação: revisar o HTML em criatives/[slug]/design-a.html antes de incluir no pacote
```

**Integração no fluxo do CEO Criativo (Story 3.2):**
O checker é executado logo após o preenchimento das variáveis no HTML, antes de salvar em `criatives/` e antes de montar o output para o CEO Master.
