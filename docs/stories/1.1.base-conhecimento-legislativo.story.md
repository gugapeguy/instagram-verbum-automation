## Dev Notes

**Contexto crítico:**  
Esta é a story fundacional do projeto — tudo que o CEO Conteúdo gera depende do que for criado aqui. Greenfield dentro de um projeto brownfield: não há padrão anterior. O @dev DEVE estabelecer o padrão de formato que todas as stories subsequentes usarão.

**Formato padrão de documento legislativo a adotar:**
```markdown
# [Tema]

**Última atualização:** [data]
**Fonte principal:** [URL]
**Pilar editorial:** [Pilar N — nome]

## Resumo Executivo
[2-3 parágrafos explicando o tema de forma clara para empreendedor leigo]

## O que muda para...

### MEI
[impacto específico]

### PJ (demais regimes)
[impacto específico]

### PF
[impacto específico]

## Vigência e Prazos Importantes
[datas relevantes em ordem cronológica]

## Fontes
- [fonte 1 com URL]
- [fonte 2 com URL]
```

**Estrutura de pastas a criar:**
```
knowledge/
├── sources.md
└── legislacao/
    ├── reforma-tributaria.md
    ├── simples-nacional.md
    ├── calendario-fiscal-2026.md
    └── esocial-trabalhista.md
```

**Restrição de conteúdo (Brand Guide Seção 8):**
- NUNCA dar orientação fiscal categórica ("você DEVE fazer X")
- SEMPRE contextualizar como informação de referência educativa
- Linguagem clara, sem juridiquês
- Conteúdo é injetado no CEO Conteúdo — deve ser legível pelo modelo

**Integração com Épico 2:**
Os documentos criados aqui são consumidos diretamente pela Story 2.2 (CEO Conteúdo). O formato padrão facilita a injeção no contexto do LLM. Nunca usar tabelas complexas — preferir listas e parágrafos curtos.

**Nota sobre fontes:**
Reforma Tributária: lei vigente é a LC 214/2024, embasada na EC 132/2023. Cronograma vai até 2033. Para calendário fiscal 2026: os vencimentos do DAS/MEI são geralmente dia 20 de cada mês. DCTF mensal: último dia útil do segundo mês seguinte ao fato gerador.
