**Como** CEO Operações,  
**quero** um módulo que faça upload dos PNGs gerados localmente ao Meta e retorne os `media_id` necessários para montar o post,  
**para que** a ponte entre o criativo exportado pelo CEO Criativo e a publicação no Instagram seja automática e confiável.

1. `api/upload-media.js` criado: importa config de `./config.js`

3. `uploadMultiple(filePaths)` exportada: recebe array de paths → executa uploads em sequência → retorna `[media_id, ...]` na mesma ordem dos slides
4. Estratégia de URL documentada no Dev Notes: abordagem para enviar imagem local ao endpoint do Graph API (que requer URL pública)
5. Timeout: 30 segundos por upload — rejeita a promessa com erro descritivo se excedido

8. Progresso: log para cada upload concluído — `✅ Upload slide [N/total] — media_id: [id]`

## Dev Notes

**Estratégia de upload — análise das opções:**

A Meta Graph API para Instagram requer uma `image_url` pública acessível. Para imagens locais, há 3 abordagens:

**Opção A (Recomendada para desenvolvimento):** Servidor HTTP local temporário
- Subir um servidor HTTP na pasta `criatives/rendered/` antes do upload
- URL: `http://localhost:3001/[slug]/slide-01.png`
- Parar o servidor após os uploads
- Problema: requer que a Meta possa acessar localhost (não funciona se a máquina não tiver IP público)

**Opção B (Recomendada para produção):** Hospedar PNG temporariamente em serviço de storage
- Fazer upload do PNG para um bucket S3/R2/Supabase Storage temporário
- Usar a URL pública gerada para o endpoint da Meta
- Deletar após confirmação de publicação
- Requer credenciais adicionais de storage

**Opção C:** Usar `image_data` (base64) — verificar suporte atual da API do Instagram

**Decisão:** @architect valida qual abordagem usar. Implementar a opção escolhida e documentar no código.

**Endpoint de upload Instagram:**
```
POST https://graph.facebook.com/v18.0/{instagram-user-id}/media
  ?image_url={URL_PUBLICA_DO_PNG}
  &caption={legenda}   (opcional — pode ser passado na publicação)
  &access_token={ACCESS_TOKEN}

Resposta: { "id": "media_container_id" }
```

**Nota importante:** O `media_id` retornado pelo endpoint de upload é um "container" — ele ainda precisa ser publicado via `POST /{instagram-user-id}/media_publish` (Story 4.3). O upload não publica; apenas cria o container.

**Timeout com AbortController:**
```javascript
const controller = new AbortController();
const timeout = setTimeout(() => controller.abort(), 30000);
try {
  const res = await fetch(url, { signal: controller.signal });
  clearTimeout(timeout);
  return res;
} catch (err) {
  if (err.name === 'AbortError') throw new Error('Timeout: upload excedeu 30 segundos');
  throw err;
}
```

**Retry logic — apenas erros de rede:**
```javascript
async function withRetry(fn) {
  try {
    return await fn();
  } catch (err) {
    // Só faz retry em erros de rede, não em 4xx da API
    if (err.message.includes('Timeout') || err.message.includes('network') || err.name === 'FetchError') {
      return await fn(); // segunda tentativa
    }
    throw err;
  }
}
```
