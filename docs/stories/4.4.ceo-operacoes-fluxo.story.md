## Story

**Como** CEO Master,  
**quero** um fluxo de operações documentado que execute toda a publicação automaticamente após a aprovação de Gustavo,  
**para que** a resposta "aprovado · [horário] · A" resulte em post publicado no @verbumcontabilidade sem nenhuma ação adicional.

## Acceptance Criteria

1. `skills/ceo-operacoes.md` criada documentando o fluxo completo de 6 passos obrigatórios
2. CEO Operações valida o gate de aprovação: presença dos 3 elementos (`aprovado`, horário, design A ou B) antes de iniciar qualquer ação
3. Seleção de design: Design A → exportar HTML via `api/export-html-to-png.js` → upload via `api/upload-media.js`; Design B → instrução para criação manual no Canva + upload posterior
4. Upload dos PNGs (Design A): coordena com `api/upload-media.js` → coleta `media_ids`
5. Publicação: seleciona `api/publish-carousel.js` ou `api/publish-feed.js` conforme tipo de post + horário aprovado
6. Verificação: `api/check-status.js` confirma publicação → atualiza `logs/publicacoes.json`
7. Validação de `.env` na inicialização: CEO Operações verifica se `api/config.js` inicializa sem erro antes de prosseguir; se falhar, retorna mensagem de setup ao CEO Master sem escalar ao Gustavo
8. Tratamento de erro silencioso: erros internos (upload, API, timeout) não chegam ao Gustavo — CEO Operações resolve internamente ou escala ao CEO Master com mensagem limpa
9. Relatório final ao CEO Master (formato exato documentado na skill):
   ```
   ✅ Publicado — @verbumcontabilidade
   Post: [URL do Instagram]
   Horário: [horário efetivo publicado]
   Design: [A ou B]
   ```
10. Fluxo de erro ao CEO Master (quando não é possível resolver):
    ```
    ⚠️ Publicação com problema — verificando
    Motivo: [mensagem limpa sem stack trace]
    Status: [resolvendo internamente / aguardando ação de setup]
    ```

## Dev Notes

**Fluxo de 6 passos do CEO Operações (documentar na skill):**
```
1. VALIDAR  — Verificar gate de aprovação: "aprovado · [horário] · [A ou B]"
              Verificar .env configurado (api/config.js sem erro)
2. SELECIONAR — A: HTML em criatives/[slug]/design-a.html
                B: Canva manual (fora do escopo automatizado v1)
3. EXPORTAR — Design A: api/export-html-to-png.js → criatives/rendered/[slug]/
4. UPLOAD   — api/upload-media.js → [media_id_1, media_id_2, ...]
5. PUBLICAR — api/publish-carousel.js (N>1 slides) OU api/publish-feed.js (1 slide)
              Horário: parseHorario(horarioStr) → timestamp Unix
6. VERIFICAR — api/check-status.js → logs/publicacoes.json atualizado
```

**Design B na v1:** CEO Operações documenta que Design B requer criação manual no Canva. O fluxo automatizado (upload + publicação) funciona da mesma forma depois que Gustavo ou o CEO Master fornece o PNG manualmente. Na v2, pode ser automatizado via Canva API.

**Mensagem de gate inválido:**
```
❌ Aprovação inválida — não é possível publicar
Recebido: "[mensagem recebida]"
Formato esperado: aprovado · [horário] · [A ou B]
Exemplos válidos:
  aprovado · 18h · A
  aprovado · amanhã 9h · B
  aprovado · 13/06 10h · A
```

**Mensagem de `.env` não configurado:**
```
⚠️ Credenciais Meta não configuradas
Ação necessária (Gustavo): preencher o arquivo .env na raiz do projeto
Referência: .env.example — instruções de como obter cada valor
Variável ausente: [nome da variável que falta]
```
